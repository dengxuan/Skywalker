# SKY9001: Service class must be partial

| 项 | 值 |
|---|---|
| 严重性 | Error |
| 引入版本 | 2.0.0 |
| 类别 | `Skywalker.SourceGenerators` |
| 关联 SG | 所有需要"补充成员"的 SG（DI 注册、DynamicProxy、EF Repository 等） |

## 原因

Skywalker Source Generator 通过 `partial class` 向用户类注入额外成员（构造函数、方法、字段、`[ModuleInitializer]` 等）。如果类没声明为 `partial`，编译器无法把生成的部分与用户写的部分合并，SG 即便生成了代码也无效。

为了避免"看起来注册了，运行时却找不到服务"这种最难排查的事故，Skywalker 选择**编译期硬性报错**而非沉默接受。

## 错误示例

```csharp
using Skywalker.Ddd.Application;

[ApplicationService]
public class OrderAppService  // ← 缺少 partial
{
    public Task<Order> GetAsync(Guid id) => default!;
}
```

编译输出：

```
error SKY9001: Class 'OrderAppService' marked with [ApplicationService] must be declared as partial
```

## 正确示例

```csharp
using Skywalker.Ddd.Application;

[ApplicationService]
public partial class OrderAppService  // ✅ 加上 partial
{
    public Task<Order> GetAsync(Guid id) => default!;
}
```

SG 会另在 `obj/Generated/Skywalker.Ddd.Application.SourceGenerators/...OrderAppService.g.cs` 生成同名 partial：

```csharp
// <auto-generated/>
namespace YourApp;

partial class OrderAppService
{
    // SG 注入的构造函数 / 字段 / 注册调用
}
```

## 自动修复

✅ 一键修复（Sprint 4 stretch goal）：把 `class` 改为 `partial class`。当前需手动加 `partial` 关键字。

## 适用范围

只要类被以下任一 SG 标注/约定识别，就必须 `partial`：

- `[ApplicationService]` / 命名约定 `*AppService`
- `[Repository]` / 继承 `Repository<,,>`
- `[EventHandler]` / 实现 `ILocalEventHandler<>`
- `[Intercept<T>]`（拦截器目标）

不需要 `partial` 的场景：

- 仅作为接口声明（不被 SG 处理）
- 标注但 SG 明确"只读取签名不注入成员"（极少数 SG）

## 嵌套类

嵌套类要求**所有外层类**都 partial：

```csharp
public partial class Outer       // ← 也要 partial
{
    public partial class Inner   // ← 也要 partial
    {
        [ApplicationService]
        public partial class OrderAppService  // ✅
        {
            // ...
        }
    }
}
```
